# [4.1] Implementation / Build — general coding help, rubber-ducking, and maker-portal copilots

> **Matrix cell:** [[4.1]](../ai-augmented-d365ce-activity-matrix.md#41-implementation--build) · Column **[1] General-Purpose AI Assistants** · Builds on [B.1](./b.1-initial-setup.md) + [C.1](./c.1-project-setup.md) · Column-3 counterpart: [4.3](../3-coding-assistants/4.3-implementation.md)

## What this activity is

- **Build** the components as detailed on user stories and tasks.

- **Build unit tests** were applicable or **manually perform testing** were not.

**General step by step and coding help.** General AI assistants do **not** build pro-code. What it does during implementation:

1. **Snippet-level generation and explanation** — Power Fx, JavaScript web resources, FetchXML, OData. You paste the snippet in and paste the result out.
2. **Samples and step-by-step configuration instructions** — a precise walkthrough for work done by hand in the maker portal or admin center.
3. **Design rubber-ducking** — thinking through an approach before committing to it.
4. **Maker-portal copilots** — Copilot in Power Apps / Power Automate, generative pages, row summaries. These produce configuration *inside the platform* with human-in-the-loop review.

## Activities

### Generate a FetchXML query

**AI:** Claude / Copilot Chat

```
Write a FetchXML query that retrieves all active Accounts
where the primary contact has an email on the domain "contoso.com",
including account name, primary contact name, and email.
Limit to 50 results.
```

### Generate an OData Web API request

```
Write an OData query for the Dataverse Web API to retrieve
all active opportunities with estimated revenue over 50,000
and their owning user's full name.
Format as a URL and a Postman-ready request.
```

### Generate / explain Power Fx

```
Write a Power Fx formula for a model-driven app command button that is
visible only when [field] is [value] and the current user has the
[role name] role. Explain each function used so I can maintain it.
```

### Explain an unfamiliar snippet

```
Explain what this [Power Fx / JavaScript / FetchXML] does, step by step,
and flag anything that looks like a bug or an unsupported pattern.
[paste snippet]
```

### Design rubber-ducking

```
I need to implement [requirement]. I'm considering:
(a) [option A — e.g. synchronous plugin]
(b) [option B — e.g. Power Automate flow]
Given the entity volumes and the solution inventory in this Project's
knowledge, argue each option's trade-offs (performance, ALM, licensing,
maintainability) and recommend one. Push back if both options are wrong.
```

### Data migration mapping

```
Given this source schema and target Dataverse table definition,
generate a field mapping table showing: source column,
target logical name, transformation needed (if any), and data type.
[paste schemas]
```

### Data validation script

```
Write a PowerShell script that queries the Dataverse Web API
and validates that all records migrated from the source have:
- A non-null [required field]
- A valid lookup to an existing [related table] record
- No duplicate [unique field] values
Output a summary report.
```

### Maker-portal copilot work (in-platform, not chat)

Use the platform copilots directly for low-code build; the chat assistant's role is to **draft the natural-language instruction** you give them:

```
I'm going to use Copilot in Power Automate to create a flow.
Draft the flow description I should give it, precise enough to get the
right trigger and actions on the first pass:
[describe what the flow must do]
```
